syntax reference

(hex)

Converts the preceding hexadecimal word to decimal.

1E (hex)→30

(bin)

Converts the preceding binary word to decimal.

10 (bin)→2

(up)

Uppercases the preceding word. Add a count for multiple words.

go (up)→GO

(low)

Lowercases the preceding word, or a span with (low, n).

SHOUTING (low)→shouting

(cap)

Capitalizes the preceding word, or a span with (cap, n).

bridge (cap)→Bridge

. , ! ? : ;

Re-spaces punctuation tight to the prior word. Runs like ... or !? stay glued.

boring ,what→boring, what

' '

Quote pairs hug the enclosed word or phrase with no inner spacing.

' awesome '→'awesome'

a → an

Swaps the article when the next word starts with a vowel sound or silent h.

a amazing→an amazing

# about the project

One pass.
Every rule applied.

TextFlow is a small, deliberate tool: feed it a draft littered with shorthand markup, and it returns clean, publication-ready prose. No accounts, no tracking — text in, text out.

how it works

01

Annotate your draft

Write naturally, then mark the spots that need conversion — a hex value, a shouty phrase, a loose comma — with the matching inline tag.

02

Run the pass

The frontend tokenizes your text and sends it to a Go-based engine, which resolves every tag in a single deterministic sweep.

03

Export clean prose

Copy the result or download it as plain text — spacing, casing, number bases and articles all corrected, tags stripped entirely.

built for

  • Drafting with shorthand

    Jot quick notes with casing and number-base reminders inline, and let the final pass do the tidying — no manual find-and-replace.

  • Teaching number systems

    Mark hexadecimal or binary literals with (hex) / (bin) and get an instant decimal-annotated version of any text.

  • Style-guide cleanup

    Run legacy text through the punctuation and article rules to catch the small spacing and grammar slips a spellchecker misses.

about the build

VAU

TextFlow is built and maintained by Victor Akor Ukpahiu-ojo as a focused exercise in parsing, tokenization, and rule-based text transformation. The interface you're using is a static frontend — every formatting rule on this page is executed by a Go engine running standard-library code, with no external dependencies.

The project began as a set of small text-processing utilities and grew into this single-pass formatter. It's intentionally scoped: one input, one output, one consistent set of rules — built to be read, audited, and extended.
